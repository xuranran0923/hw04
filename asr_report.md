任务三 开源语音识别实践报告

一、三种开源语音识别（ASR）工具核心特征对比

1. OpenAI Whisper

- 开源协议：MIT开源协议，免费可商用，无使用限制，适配各类学习、实验场景。

- 核心特征：支持近百种语言，中文识别效果稳定、准确率高，能精准识别长句、标点符号，适配日常朗读类音频；提供base、small、large等多种尺寸模型，可根据设备性能选择，小模型（base）运行速度快，无需高性能设备；支持命令行和Python两种调用方式，上手简单，对新手友好，无需复杂配置。

- 优缺点：优点是易用性强、识别精度高、多语言支持；缺点是本地运行依赖PyTorch环境，对部分电脑配置有轻微要求，新手易出现环境相关报错。

2. Vosk

- 开源协议：Apache 2.0开源协议，可免费用于个人和商业场景，兼容性强。

- 核心特征：极度轻量，模型体积小（仅几MB到几十MB），启动速度快，支持完全离线运行，不依赖网络；支持流式实时识别，可边说话边转写，适合嵌入式设备、低配置电脑使用；支持多语言，基础中文识别可满足简单需求。

- 优缺点：优点是轻量、离线可用、运行速度快；缺点是中文识别精度一般，对复杂句式、语气词的识别准确率低于Whisper，不适合对识别精度要求高的场景。

3. FunASR

- 开源协议：Apache 2.0开源协议，阿里达摩院开源，专注于中文语音识别。

- 核心特征：专门针对中文优化，对中文口语、方言、数字、专业术语的识别精度极高，对标工业级应用；支持标点自动预测、说话人分离，能区分多个人的语音；提供多种预训练模型，适配不同场景需求。

- 优缺点：优点是中文识别精度高、功能全面；缺点是环境配置复杂，依赖项多，需要配置特定版本的依赖库，对新手不友好，部署难度高于Whisper和Vosk。

二、选型理由

综合本次作业场景、易用性和识别效果，本次选择OpenAI Whisper作为语音识别实现方案，具体理由如下：

1.  适配作业需求：本次测试音频为剪映导出的中文朗读音频，句式规整、语音清晰，Whisper的中文识别精度完全能满足需求，识别结果可与原文高度一致；

2.  易用性强：无需复杂环境配置，即使本地运行出现报错，也可通过网页版快速完成识别，高效适配新手操作，能快速完成“语音→文字”的实验闭环；

3.  适配课程场景：文档丰富、社区支持完善，调用方式简单，可轻松写出可运行代码，符合课程作业对“可复现性”的要求。

三、可运行代码及安装运行说明

1. 可运行Python代码

import whisper

# 加载Whisper base模型（轻量、速度快，适配新手电脑）
model = whisper.load_model("base")

# 识别剪映导出的音频文件（替换为自身音频文件名及路径，如：C:\Users\Administrator\Desktop
audio_path = "配音.mp3"
result = model.transcribe(audio_path, language="zh")  # 指定语言为中文，提升识别精度

# 输出识别结果并保存到文本文件
print("语音识别结果：\n", result["text"])
with open("识别结果.txt", "w", encoding="utf-8") as f:
    f.write(result["text"])

2. 安装及运行命令

- 安装依赖命令（CMD中输入，一键安装）：

pip install openai-whisper ffmpeg-python

- 运行代码命令（CMD中输入，需确保音频文件路径正确）：

python asr_whisper.py

四、实验记录

1.  测试素材：剪映导出的中文朗读音频，使用剪映内置“译制片男声”音色，文件格式为MP3，时长约1分钟，语音清晰、语速均匀，无杂音。

2.  实验环境：Windows 11操作系统，Python 3.10版本，因本地运行Whisper出现DLL动态链接库初始化失败报错，改用Whisper网页版（https://huggingface.co/spaces/openai/whisper）完成识别，识别过程无需配置环境，上传音频即可生成结果。

3.  识别结果：

为何输入几个词，电脑就能绘出逼真图像、写出动人诗文？这源于生成式人工智能的魔力。传统 AI 是 “判断者”，能分辨猫与狗；而生成式 AI 化身 “创作者”，通过学习海量文本、图像数据，掌握内容规律，再依据提示词生成全新内容。它像全能实习生，帮你高效搞定写作、设计等工作。虽存在幻觉与偏见，但它正重塑人机协作模式，成为激发创意的伙伴，掌握与 AI 对话的能力，也是我们的全新起点。

4.  结果分析：识别结果与原文完全一致，无错别字、无漏句，标点符号识别准确，Whisper的中文识别精度符合实验要求，成功完成“语音→文字”的转换。

5.  问题及解决方法：本地运行Whisper时出现“OSError: [WinError 1114] 动态链接库(DLL)初始化例程失败”报错，排查后发现是PyTorch环境与电脑配置不兼容，为高效完成作业，改用Whisper网页版，操作简单且识别结果一致，完美解决环境报错问题。
