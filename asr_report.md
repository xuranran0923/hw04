任务三 开源语音识别实践报告

一、三种开源语音识别（ASR）方案对比
本次结合个人电脑配置（Intel i5-8265U、8GB 内存、无高性能 GPU、Windows 64 位），调研 3 种主流开源语音识别工具，具体维度说明如下：
1. OpenAI Whisper
许可协议：MIT 开源协议，可免费商用。
语言支持：支持 99 种语言，包含中文普通话，但方言支持有限。
模型体量：提供多档模型（tiny/base/small/medium/large 等），体积从几百 MB 到 2GB + 不等。
推理速度：小模型推理速度较快，大模型速度显著下降。
实时支持：不支持原生实时流式识别，需自行开发二次逻辑。
部署难度：安装流程简单，通过 pip 即可完成，但对显卡驱动有一定要求。
PC 适配性：在本人电脑上安装后，因显卡兼容性问题，出现 DLL 动态链接库初始化失败报错，无法正常运行，最终放弃使用。
2. Vosk
许可协议：Apache 2.0 开源协议，可免费商用。
语言支持：支持多语言及中文方言，对普通话和常见方言优化效果良好。
模型体量：模型超轻量，中文基础小模型仅需约 50MB，部署体积极小。
推理速度：推理速度极快，纯 CPU 环境下即可实现秒级识别。
实时支持：原生支持实时流式识别，可直接接入麦克风输入。
部署难度：部署难度极低，纯 CPU 运行，无复杂系统依赖，新手友好。
PC 适配性：完美适配本人低配置轻薄本，安装后运行稳定，无卡顿、报错，可完成识别任务。
3. FunASR
许可协议：Apache 2.0 开源协议，可免费商用。
语言支持：针对中文及方言深度优化，支持普通话、粤语等多种语言场景。
模型体量：提供轻量化模型与大模型多档可选，基础模型体积约 1GB。
推理速度：轻量模型推理速度快，大模型推理速度中等。
实时支持：原生支持实时流式识别，适合实时语音转文字场景。
部署难度：环境配置较为复杂，需安装多类依赖库，部署难度中等。
PC 适配性：虽功能强大，但对内存要求较高，本人 8GB 内存环境下运行易出现资源占用过高、卡顿等问题，因此放弃选用。
二、工具选型理由
本次实验最终选用Vosk 轻量中文模型，核心选型依据如下：
硬件适配性最优：纯 CPU 运行，不依赖高性能 GPU，内存占用极低（运行时占用 < 500MB），完美适配本人低配置轻薄本，从根本上避免了 Whisper 的显卡报错问题和 FunASR 的内存卡顿问题。
部署门槛最低：仅需通过pip install vosk一条命令即可完成安装，无需复杂的环境配置，新手操作难度低，符合作业快速落地的要求。
需求匹配度高：原生支持 WAV 音频格式读取，可直接调用任务二导出的配音音频文件，完成 “音频→文本” 的核心转换流程，完全满足作业开源工具使用的需求。
开源合规性：采用 Apache 2.0 协议，无商用限制，完全符合作业对开源工具的规范要求。
功能可拓展性：支持实时流式识别，具备拓展为麦克风实时输入识别的潜力，符合作业鼓励拓展实践的要求。
三、可运行代码（hw04 目录下）
1. 代码文件：asr_vosk.py
from vosk import Model, KaldiRecognizer
import json
import os


# 任务二导出的配音音频路径
AUDIO_FILE = "jianying_voice.wav"
# 识别结果保存路径
RESULT_FILE = "asr_result.txt"


print("正在加载Vosk中文轻量模型...")
model = Model("model")
rec = KaldiRecognizer(model, 16000)

# ===================== 音频识别 =====================
try:
    # 读取音频文件
    with open(AUDIO_FILE, "rb") as f:
        audio_data = f.read()
        # 执行识别
        rec.AcceptWaveform(audio_data)
        result = json.loads(rec.FinalResult())
        recognized_text = result["text"]

    # ===================== 结果输出与保存 =====================
    print("\n" + "="*50)
    print("✅ 语音识别任务完成！")
    print("识别结果：\n" + recognized_text)
    print("="*50)

    # 保存结果到文件
    with open(RESULT_FILE, "w", encoding="utf-8") as f:
        f.write(recognized_text)
    print(f"\n结果已保存至：{RESULT_FILE}")

except FileNotFoundError:
    print(f"\n❌ 错误：未找到音频文件 {AUDIO_FILE}，请确认文件路径正确！")
except Exception as e:
    print(f"\n❌ 识别过程出现错误：{str(e)}")

4. 结果分析
流程完整性：实验全流程跑通，成功实现语音到文本的转换，完成作业核心要求，无技术报错。
精度差异原因：识别结果与原文本差异较大，核心原因是选用的 Vosk 轻量模型参数少、体积小，对长文本、连续语音的捕捉能力有限，仅能识别出部分虚词，符合轻量模型的特性。
工具优劣势总结：
优势：低资源设备适配性强、离线可用、部署简单、运行稳定；
劣势：识别精度有限，不适合长文本高精度识别场景。
优化方向：若需提升识别精度，可更换 Vosk 大体积中文模型，或在硬件升级后使用 Whisper medium/large 模型；同时可优化音频质量（降低语速、减少背景噪音），提升识别效果。
